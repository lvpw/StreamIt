This page is the home of StreamIt on Cell, including the StreamIt backend for Cell, a lightweight streaming runtime library, and a dynamic scheduler. Here is what we've designed and implemented:

  • A runtime framework that implements a schedule and maps computation to SPEs and coordinate data transfers between SPEs as necessary. This component consists of a runtime library with a streaming intermediate format (to describe the filters and data channels) and a dynamic scheduler which dynamically determines a mapping of filters to SPEs. The library admits statically defined schedules as well.
  • A compiler backend that performs code generation for the runtime library: it provisions the StreamIt stream graph for the intermediate representation used by the runtime library, and optionally outputs a schedule for the runtime to implement.

Some details of this work appear in our dasCMP 2007 paper. More details can be found in David Zhang's MEng thesis.

We have a pre-release package of the runtime system available for download.

The distribution includes the runtime library, the scheduler, and some examples. Since this is a pre-release (but in documented and reasonably good standing) you may have some questions that are faster for us to answer than for you to dig around and uncover on your own. Feel free to contact us if you run into problems.

The library was developed and tested with Cell SDK 2.0, although it will compile with SDK 3.0 or later (we have not tried the latest SDK release). It was tested on both Yellow Dog and Fedora Core Linux distributions running on a PS3 and an IBM QS21 Cell blade center.

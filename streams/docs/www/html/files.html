  • Stable Snapshot Release, August 19, 2005.
        Download: [binary .tar.gz]  [source .tar.gz]
      Recommended download. This snapshot has new features, bug fixes, and possibly new bugs compared to the initial public release. Major features are as follows, please contact us if you have any questions:
      • Faster and more memory-efficient Java library (up to 10X).
      • Faster buffer management in uniprocessor library (up to 2X).
      • JPEG decoder/encoder application (supported in Java library only).
      • Limited support for dynamic rates and teleport messaging in Java library.
      • Undocumented snapshot of cache optimizations and cluster backend.

  • Stable Public Release, October 20, 2003.
        Download: [binary .tar.gz]  [source .tar.gz]
      Our first public release.

    Unpack the tar file, and read the INSTALL file for installation instructions. Note that while these releases work for most well-formed programs, there may be issues with the compiler we have not found yet, and there is not much error checking in the frontend. Read the README file in the release for more details on known shortcomings.

    Dependencies

    • RAW infrastructure tools. Set these up before trying to use StreamIt if you are compiling for RAW. Make sure the TOPDIR environment variable points to your StarSearch directory.

    • JIKES Java Compiler (source release only). Also, a Java Virtual Machine is required to run the compiler.

    • ANTLR LL(k) parser generator (source release only). This is needed by the StreamIt frontend; if you are using the source release, the ANTLR jar file must be in your Java class path to read in StreamIt files.

    • FFTW FFT library. Download and install FFTW before building the uniprocessor C library, if you are using the uniprocessor target. Use version 2.1.x of FFTW, not 3.0, and compile with --enable-float --enable-type-prefix.

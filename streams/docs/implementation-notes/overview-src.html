]> StreamIt Source Code Roadmap

Package at.dms contains the DMS's Kopi compiler for Java with the additions that form the StreamIt compilers. Most of StreamIt the modifications for the compilers are in at.dms.kjc, but at.dms.util also contains some StreamIt code. Note that the parser has been changed from parsing Java to parsing a variant that requires sizes to be declared for all arrays.

Package streamit contains

  • the code for the StreamIt interpreter a.k.a. the library back end streamit.library.
  • streamit.frontend which contains lexical analysis and parsing of StreamIt code and conversion either into Java code for the library back end, or into a variant of Java code annotated with sizes for all array declarations suitable for the compier back ends.
  • streamit.eclipse contains eclipse addons for graphical program development and debugging.
  • streamit.scheduler2 contains scheduling code that works both in the library and in the compiled back ends. The scheduling relies on streamit.library.iriter for iterating in the library and at.dms.kjc.iterator for iterating in the compiler back ends.

Package com.jgraph appears obsolete and should be replaced with org.jgraph. This third-party package is used only by the graph editor eclipse add-on.
